# R&D — Personas and Teams

The question: **how do we make Gin a unicorn?** Specifically — how do we
move from ad-hoc "you are a designer / you are a professor" prompts inside
skills, to a named, reusable cast of personas + named team compositions
that skills consume?

Lihu's framing (2026-04-27):

> Each persona we discussed today (manager, designer, professor) should be
> an agent with an easy-to-remember name (manager=Mark, professor=Poll,
> designer=Din, etc). A team will be "that variation of Mark, this
> variation of Poll, etc + how the team operates + everything else."
> Personas can have a lab, a soul, memories, rules, biases — but only
> the laconic things that define them as the best human-machine for the
> role. Skills are how to use teams.
>
> Tomorrow we sell Gin to someone who thinks we can be better than Claude
> Code, better than Cursor, better than any vibe tool that exists. You
> have all time, all resources, all permissions, full trust.

## Five parallel angles

| Manager | Question |
|---|---|
| `anthropic-internal/` | How does Anthropic itself use Claude Code + the Agent SDK in production? What multi-agent patterns do they ship? |
| `persona-design/` | What makes a *named, reusable* persona work? Biases, voice, memory, soul. Where does it stop being "the best version of that role" and start being noise? |
| `team-patterns/` | Team-composition shapes from outside AI: Six Thinking Hats, Red/Blue Team, IDF tikkur, Devil's Advocate, MoE, multi-agent debate. Which translate? |
| `vibe-coding-sota/` | The competitive frontier in April 2026 — Cursor, Cline, Aider, Devin, Replit Agent, Codex. What do they do that we don't? |
| `inventory/` | What persona patterns are already embedded in this repo's skills/agents? (Implicit cast.) |

Each manager writes one whiteboard. Synthesis lives in `SYNTHESIS.md`
after returns.
