## 1. "The lack of persistent understanding is glaringly missing"

**Chris Baum → Guy + team | Email thread "Keep an eye on?" | Jan 7–8, 2026**

> *"I'm SO GLAD that this is a thing. The lack of persistent understanding is just glaringly missing from AI systems. I don't get why they are so obsessed in the video with the agents using the context graph, tho. It'll be just as important (maybe more?) for humans. It's a feature we've been missing since, well, forever. Can you imagine what it looks like when there is an actual shared, referenceable mental model for both people and machines???"*

Guy's reply (Jan 8):
> *"Rather than architecting a mental model for the agent we can let the agent choose a mental model / context graph. They will make something that is richer and works better for them."*

Chris's follow-up (Jan 8):
> *"One clarification: the provocation was to let the agents create the context graph 'of record'. They aren't 'choosing' that mental model or making different ones for the same context, they are emergently building the context graph for everyone. It's not just for themselves, but for all of the AI and for the people. Having ways for humans to contribute data elements (e.g. confirm decisions, etc.) would speed up the process, but the humans should not be crafting a schema manually. That schema should be 'discovered' by the agents."*

## 6. Guy's workaround — manually baking in "keep context in working memory"

**Guy Levit → team | Email "Quick helpful fix: augmenting system prompt" | Feb 3, 2026**

Guy found he got "MUCH BETTER answers" by starting every session with a manual prompt. He asked Lihu to bake it in immediately:

## 7. Context compression hitting mid-session — Guy documents it in real time

**Guy Levit → team | Email thread "Testing Effi with a large project" | Apr 12, 2026**

Guy shared a live Effi session excerpt where the context limit visibly broke things. The session itself explained what happened:

> *"Rather than hard-failing, the system compresses prior messages into a summary (like the one at the top of this session). That summary is lossy — exact transcript text, specific quotes, and intermediate tool results don't survive it. That's why some tasks (Andrea's summary, the customer segmentation) were 'completed' in tool calls but the actual response never landed — the context filled up before I could write it."*
>
> *"Session breaks between heavy tasks — starting a fresh session for each major analysis keeps the buffer clean."*
>
> *"The 208-meeting corpus is genuinely large. The tools are working, but the session needs to be managed like a resource."*
