## Human side

(Reconstructed from "We do everything manually, middle, like pre-game. We do everything just to make it work and then to think." — Lihu, this session. Open-to-empty for further refinement.)

Pre-game = the manual phase before the system exists. We do it by hand to make it work, *then* we think about how to systematize. We don't design the system from a chair.

## Gin side

The four manually-written principles in `usegin/zettel/principles/` are pre-game artifacts. The 4 zettels seeded by hand in this session are pre-game artifacts. The Wispr corrector dictionary seeded with this session's actual corruptions is pre-game.

The discipline: **only systematize what we've actually done by hand at least once.** Don't pre-build affordances for behaviors we haven't tried. Pre-built affordances pretending to be use-grown ones are a known failure mode in PKM (Anthropologist whiteboard, P-DESTINATION pattern).

This zettel is itself a pre-game artifact for the `_zettel_custom_` tool — we're writing zettels by hand right now, in the file system, before there's a tool to write zettels in. When the tool exists, it should preserve what we learned doing it manually, not erase it.
