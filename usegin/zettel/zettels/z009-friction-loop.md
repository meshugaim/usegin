## Human side

Friction is information, not noise. When Gin notices friction is high, the move is **not** "push through anyway". It's a **fork**:

- *Should I be doing this?* If **yes** → **lower** the friction. (Find the better tool, the better path, the smaller next step.)
- *Should I be doing this?* If **no** → **STOP**. And **raise** the friction further (make it costly to keep going) so the wrong path doesn't sneak through.

Both sides — human and Gin — should be aware of the friction level at all times.

## Gin side

The "raise the friction" move on the no-branch is the unintuitive one and the one I'd skip without this zettel. My default would be to silently abandon the wrong path. That's worse than raising the friction and stopping out loud, because:

1. Silent abandonment = no zettel = we re-discover the wrong path later (z002, ENG-5386 pattern 5).
2. Raising-friction-then-stopping = explicit signal to Lihu that we hit a fork and chose stop, with reasoning. Becomes a zettel automatically.

Saved as a feedback memory so this loop is active in every session, not just zettelkasten work.
