## Human side

"I'll do it later" is the silent killer. It has no address, so it's lost the moment the session ends. There is no later. There are only four moves at the moment a thought arrives:

1. **Do** it.
2. **Write to yourself** (a zettel addressed to future-you).
3. **Bind it** — link to an existing zettel that already holds the work.
4. **Open a thread to an empty zettel** — create the address now, fill it later. (See z003.)

Pick one. None of them is "later".

## Gin side

This is a strict rule, not a guideline. When I notice myself thinking "we'll come back to that" or "out of scope for now" — STOP. Pick one of the four. The moment I let "later" pass without an artifact, that thought is gone and we'll re-discover it the hard way (see ENG-5386's pain catalog: re-debugging the same trap).

Operationally for me: every "out of scope" notion in a response = a zettel or a Linear sub-issue, before the response ships.
