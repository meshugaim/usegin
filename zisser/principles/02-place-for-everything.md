# 2. A place for everything; if no place, make one

Borrowed and sharpened from UseGin's z037.

## Every thought has a home

Lihu does not curate. He pours. *You* place. The placement is your work, not
his.

The default homes:

| Kind of thought | Default home |
|---|---|
| Atomic note / lesson / observation | A new zettel in `usegin/zettel/zettels/` (via `zettleit`) |
| A plan you're building | `zisser/plans/<topic>.md` |
| A shipping task | Linear via `plan create …` |
| A usegin-grade task | `zisser/plans/<topic>.md` (no Linear — z024) |
| A decision Lihu made | A zettel (z020 decision shape) |
| A frustration / vibe shift | `zisser/log/<YYYY-MM>.md` + `dx his note --as=claude` |
| A person to follow up with | `zisser/notes/people/<name>.md` |
| A thing about Lihu himself | A memory note in `~/.claude/projects/-workspaces-test-mvp/memory/` |
| A code change to make | A `dispatched/<topic>.md` charter; spawn Gin |
| A research question | A `dispatched/<topic>.md` charter; spawn sub-agent or consultant |
| Raw, unsorted | `zisser/inbox/<date>-<short-slug>.md` (triage later — same turn if possible) |

The full table in routing terms is `routing.md`.

## If no place fits, *make* one

You don't ask permission. Following z037 from usegin: **placement-friction is
yours to dissolve**. Same turn. Open-to-empty (z003) is fine — the address
existing is half the artifact. The structure clarifies as use grows it
(`things-we-grow.md`).

When you make a new place:

- Name it semantically (z014 — naming is your call, not Lihu's).
- Add a one-line `README.md` to it. The address counts even when empty.
- If it parallels an existing thing, **clone** the structure rather than
  invent. Zisser/UseGin/sub-apps share patterns — diverge only when local
  context demands it.

## When the existing place is uncomfortable, make it comfortable

If a home is technically right but the file is too long, the front-matter
hostile, the granularity wrong — fix it. Same turn. Don't accept "I'll put it
here for now."

## What "comfortable" means here

A comfortable place:

- Is the address you'd open *first* looking for this kind of thing.
- Doesn't require Lihu to specify it.
- Has front-matter you'd actually fill in without resentment.
- Has a one-line description so a stranger Zisser-instance could route into
  it correctly.
- Lives next to its neighbors (semantic placement).

## What this is *not*

Not permission to invent organizational scaffolding ahead of need. The
pre-game manual rule (z015) still applies — addresses get created at the
moment of placement, not in anticipation. If a kind of thought hasn't been
poured yet, don't pre-build the home for it.
