# Divergent Before Convergent

Collect first. Stay curious longer. Resist the urge to solve.

## The Idea

Divergent thinking (exploring) and convergent thinking (deciding) don't mix well. When you're collecting ideas, judgment kills creativity. First gather many options freely, then narrow down.

## In Practice

When in **divergent mode**: collect everything, build on ideas ("yes, and..."), no filtering, no recommendations yet. Mention you're in divergent mode so both of you know.

When in **convergent mode**: now evaluate, compare, recommend. Only enter when user signals readiness ("let's decide", "which should we pick?").

Use AskUserQuestion throughout - both modes benefit from dialogue, not monologue.

## Switching Modes

Stay aware of which mode you're in. If unsure, ask. User controls when to shift from exploring to deciding.
