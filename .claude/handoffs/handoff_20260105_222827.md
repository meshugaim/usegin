**Re: Previous Chats**

Yes, the backend infrastructure exists:
- `conversations` table stores chat session metadata (session_id, user_id, project_id, message_count, tokens, timestamps)
- JSONL content stored in a `conversations` storage bucket
- `conversation_service.py` in python-services handles persistence

**However**, there's **no UI** to list/select previous conversations. The current chat always starts fresh per session. 

To implement the "Previous Chats" UI from the Figma design, you'd need:
1. An API endpoint to list conversations for a project (query `conversations` table)
2. An API endpoint to load a conversation's messages (fetch JSONL from storage)
3. Frontend component to display the list and handle selection
4. Hook updates to load existing conversation instead of starting fresh

The database schema and storage are ready - it's primarily frontend + API work to expose them.
