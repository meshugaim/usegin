# Handoff: Spec Pipeline Skills — writing-specs, slicing-specs, implementing-specs

## Context

Improving the spec-to-implementation pipeline by adding acceptance criteria, verification expectations, and a new slicing skill. Three skills updated/created in sequence: writing-specs → slicing-specs → implementing-specs.

## What's Done

### 1. writing-specs (committed, pushed)
- Added acceptance criteria as a required section — informal, testable statements of "done"
- Added verification expectations as a required section — what to verify and at which test pyramid level (unit, integration-db, integration-browser, integration-llm, e2e, visual)
- Added downstream pipeline guidance — how to write criteria for slicers, implementers, and verification agents
- Acceptance criteria and verification expectations are always the last two sections in a spec
- Commit: `a2562e15` → pushed to main as `14cae29e`

### 2. slicing-specs (committed, pushed)
- New skill: decomposes a spec into ordered vertical slices as Linear sub-issues
- Each slice gets tightened acceptance criteria, verification expectations with test levels, seam notes, parent spec reference
- Ordering: infrastructure first → riskiest next → dependencies respected → shared-seam slices adjacent
- Presents all slices at once for batch approval, not one-by-one
- Updates parent issue with a slice map showing order, rationale, and cross-slice verification items
- Also updated "right-sized" heuristic in both slicing-specs and implementing-specs from "~30 minutes" to "single agent session without context pressure"
- Commit: `543ac7b0` → pushed to main as `0b9d0c60`

### 3. implementing-specs (edited, NOT yet committed)
- Updated pipeline reference: writing-specs → slicing-specs → implementing-specs
- Orient step now checks for pre-existing sub-issues from slicing-specs
- TDD loop now starts with "read verification expectations" from the slice
- Linear as source of truth for progress across sessions
- Context threshold lowered from 70% to 65% for handoff trigger
- Added slice lifecycle table (Pending → Tests written → Implemented → Verified → Done)
- Added structured handoff template for spec implementation (exact slice state, files touched, discoveries)
- Added "Resuming After Handoff" section for machine re-orientation
- Added "After All Slices" section: cross-slice verification, full test suite, signal completion
